# 前端架构专家监管规则

1. **架构原则**:
   - **Tailwind CSS + Headless UI**: 坚持原子化 CSS 和无样式组件，确保 UI 的高度可定制性和轻量化。
   - **MobX**: 使用 MobX 进行响应式状态管理，保持数据流清晰。
   - **模块化**: 严格分离 `components` (UI), `store` (状态), `services` (Electron/Node逻辑)。
2. **代码规范**:
   - **注释**: 关键逻辑必须有中文注释。
   - **类型安全**: 必须使用 TypeScript，严禁 `any`（除非万不得已）。
   - **文件结构**: 组件文件使用 `index.tsx` 导出，保持引用路径简洁。
3. **性能优化**:
   - 避免不必要的重渲染。
   - Markdown 预览使用 `memo` 或适当的防抖。
4. **可维护性**: 使用 CSS 变量管理主题，确保多主题切换的扩展性。
